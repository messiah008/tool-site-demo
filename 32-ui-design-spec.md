## 一、核心理念:一进来就是完整产品

### 1.0 反面教训(2026-07-14 新增:布局反直觉)

| 错误做法 | 用户感受 | 出现工具 |
|---------|---------|---------|
| **触发按钮在上方,结果在下方** | "点了之后视线要往下找结果,违反直觉" | 易经起卦/抽奖/随机数生成 |
| 转换按钮在中间,结果在下方(非实时) | "每次都要点一下才能看到结果" | base64-encode/morse-code |

**正确做法**:结果展示区永远是视觉焦点(顶部醒目位置),触发按钮放在结果区**下方**或**旁边**,点击后结果在原位刷新。这样用户的视线焦点不需要移动,符合"占卜/抽签/抽奖"类工具的直觉。

### 1.1 反面教训(我们踩过的坑)

| 错误做法 | 用户感受 | 出现工具 |
|---------|---------|---------|
| 几个空输入框 + 一个按钮 | "太简陋了,不会留下" | 个税/房贷初版 |
| 需点"计算"才出结果 | "点了才知道有什么功能" | 所有计算器初版 |
| 功能藏在结果区(v-if折叠) | "看不到功能,以为没做" | 房贷明细表/提前还款 |
| 新功能只改逻辑不改界面 | "界面没变化" | 万年历初版 |
| 数据只有几条 | "内容太少,没价值" | 谜语8题/成语45条 |

### 1.2 核心原则

**用户进页面第一眼,就能看到完整的产品功能,而不是"填表→出结果"的工具。**

7 条铁律:

1. **默认值 + 自动计算** — 进页面就有预设值和结果,不需要用户先填再点
2. **结果区始终可见** — 结果不是"点击后才出现",而是"一进来就在"
3. **功能可见性** — 所有功能(明细表/对比/提前还款)都在首屏可见或明确标注"可展开"
4. **可视化优先** — 用图表/进度条/卡片替代纯数字
5. **对标成熟产品** — 照抄 AuxTool/房天下/bmcx 的功能,不自创
6. **数据量要够** — 谜语100+/成语500+,太少用户来了也留不住
7. **触发按钮在结果区下方/旁边,不在上方** — 结果展示区是视觉焦点,点击后结果在原位刷新,视线不移动

## 二、页面结构规范

### 2.1 标准布局

```
┌─────────────────────────────────────┐
│ ToolLayout(面包屑+标题+互链+关于)    │
├─────────────────────────────────────┤
│ Tab切换(如有多个模式)               │  ← 如:个税[月薪|年终奖]
├─────────────────────────────────────┤
│ 输入区(网格布局,预设默认值)          │  ← 2列网格,非单列堆叠
├─────────────────────────────────────┤
│ 结果区(一进页面就可见)               │
│  ├ 核心数字(大字+渐变背景)           │
│  ├ 可视化(进度条/占比条/对比图)      │
│  ├ 明细卡片(网格)                   │
│  └ 可展开区块(明细表/高级功能)       │
├─────────────────────────────────────┤
│ 提示/建议(配色块)                   │
└─────────────────────────────────────┘
```

### 2.2 输入区规范

| 规则 | 说明 |
|------|------|
| 网格布局 | 2列(桌面)/1列(移动),非单列堆叠 |
| 预设默认值 | 每个字段都有合理默认值(如月薪15000/贷款100万) |
| 实时计算 | 输入即计算(@input/@change),不需要点按钮 |
| 下拉选择 | 枚举值用 select(如公积金比例5-12%),不用手动输 |
| 单位标注 | 标签后加小字单位(如"元/月""%""万元") |
| 提示文字 | 复杂字段加 hint(如"房贷1000/子女1000-2000") |

### 2.3 结果区规范

| 规则 | 说明 |
|------|------|
| 始终可见 | 用 v-if="result" 但 onMounted 就计算,result 一进来就有值 |
| 核心数字 | 大字(32px+)+渐变背景卡片,一屏内最醒目 |
| 可视化 | 用进度条/占比条/对比图替代纯数字表格 |
| 明细卡片 | 4列网格,每卡一个指标(如税前/社保/税率/应纳税额) |
| 可展开 | 大量明细用"点击展开"(如还款明细表/提前还款) |
| 颜色语义 | 绿色=到手/成功,红色=个税/支出,黄色=利息/警告 |

### 2.4 Tab切换规范(多模式工具)

如工具有多个模式(个税:月薪/年终奖):
- Tab 在顶部,2-3个
- 每个 Tab 有图标+文字
- 切换时自动计算,显示对应结果
- 默认激活第一个 Tab

## 三、各类工具的设计模板

### 3.1 计算器类(个税/房贷/利息/BMI/退休)

```
输入区(2列网格,预设值) → 自动计算
结果区:
  ├ 核心数字(到手¥/月供¥)大卡片
  ├ 构成可视化(占比条:各部分占比)
  ├ 明细卡片(4列:税前/社保/税率/应纳税)
  ├ 可展开:明细表(前12期/12月)
  └ 可展开:高级计算(提前还款/年终奖对比)
提示(配色块:💡建议文字)
```

### 3.2 查询类(身份证/手机/成语/诗词)

```
输入区(搜索框,可空) → 自动查询或点查询
结果区:
  ├ 结果卡片(网格,每条一个卡片)
  ├ 高亮关键信息(如省市区/运营商/释义)
  └ 空结果提示("未找到,试试其他关键词")
分页/加载更多(如数据多)
```

### 3.3 日历/网格类(万年历/节气)

```
工具栏(年月切换+今天按钮)
双栏布局:
  ├ 左:网格(固定行高,不用aspect-ratio)
  └ 右:详情面板(点击单元格显示)
移动端:单栏(网格上+详情下)
```

### 3.4 趣味类(谜语/歇后语/名言/对联)

```
大卡片(居中,渐变背景)
  ├ 内容(谜面/歇后语/名言)
  ├ 答案(可展开"查看答案")
  └ 下一个(随机切换)
计数("共 N 条")
分类筛选(如对联:春联/婚联/寿联)
```

### 3.5 详情类(生肖/星座/朝代/百家姓)

```
输入区(年份/月日/姓氏选择)
结果区:
  ├ 头部(大图标+名称+一句话)
  ├ 性格/描述(段落)
  ├ 优点/缺点(双列,绿色/红色卡片)
  ├ 配对/幸运(双列,蓝色/黄色卡片)
  └ 名人/大事(标签云)
```

### 3.6 生成/占卜/抽奖类(密码/UUID/易经/抽奖/随机数) — v2.1 新增

**核心原则**:结果展示区是视觉焦点,触发按钮在下方。**所有"点击生成XX"的工具都适用此模板**(密码/UUID/易经/抽奖/随机数/口令卡等)。

```
┌─────────────────────────────┐
│ 舞台/结果区(始终居中醒目)     │  ← 视觉焦点,大字+渐变背景
│  ├ 空状态:大符号+提示文字    │
│  ├ 进行中:滚动动画(摇卦/掷骰)│
│  └ 结果:大数字/卦象/中奖号码 │
├─────────────────────────────┤
│ 配置区(长度/字符集/数量等)   │  ← 在舞台下方或右侧
├─────────────────────────────┤
│ 触发按钮("🔑 生成"/"🎲 抽奖") │  ← 在配置区下方
│  └ 旁边:清空/再生成         │
├─────────────────────────────┤
│ 详情区(破解耗时/算法对比/统计)│
└─────────────────────────────┘
```

**关键**:结果舞台必须**永远占位**(空状态也有大符号提示),不能是"按钮上方一片空白,点完按钮才填上"。空状态应该比按钮还显眼,这样按钮点击后视线焦点不需要移动,结果在原位刷新。

| 元素 | 规范 |
|------|------|
| 结果舞台 | 顶部,大尺寸(结果字 32-64px),渐变背景,永远占位 |
| 空状态 | 大图标(56px+)+ 简短提示,比按钮更显眼,不能是一片空白 |
| 配置区 | 在舞台下方或右侧,不挤占视觉焦点 |
| 触发按钮 | 配置区下方,主按钮+次按钮(清空/再生成) |
| 滚动动画 | 触发后 600-800ms 结果快速滚动,模拟"摇卦/掷骰/生成中"过程 |
| 结果出现动画 | scale+opacity 渐入,强化"结果显现"仪式感 |
| 计数 | "已生成N次"在底部 |

### 3.7 实时转换类(JSON/CSS/Base64/Morse) — v2.1 新增

**核心原则**:输入即转换,不需要"转换"按钮。

```
输入区(textarea)
工具栏(模式切换:美化/压缩 | 编码/解码)  ← Tab切换,不是"触发"按钮
结果区(实时显示,一进页面就有默认值转换)
```

| 元素 | 规范 |
|------|------|
| 输入区 | textarea,预设默认值,@input 实时触发 |
| 工具栏 | 模式切换Tab + 复制/清空按钮(不是"转换"按钮) |
| 结果区 | onMounted 用默认值转换一次,一进来就有结果 |
| 错误显示 | 输入不合法时显示错误,但不影响结果区位置 |

**禁止**:不要做"输入→点转换按钮→结果"的流程,这是反直觉的。

### 3.8 通用审美规范 — v2.2 新增

**核心原则**:工具不是表单,是"一眼看出能做什么"的产品。所有工具首屏必须有视觉重心,不能是"几个空输入框+一个按钮"的工具式样貌。

#### 3.8.1 首屏视觉重心

每个工具首屏必须有**一个**最显眼的元素,用户视线落点:

| 工具类型 | 视觉重心 |
|---------|---------|
| 计算器 | 大数字结果卡片(渐变背景,32-36px) |
| 生成器 | 大舞台(结果展示区,空状态也有大符号) |
| 转换器 | 暗色代码框(语法高亮,带行号) |
| 查询器 | 结果卡片网格(每条一个卡) |
| 日历 | 大网格(占首屏 60%) |

**反面**:几个空输入框 + 一个按钮 = 工具式样貌,用户觉得"太简陋,不会留下"。

#### 3.8.2 空白处理

| 规则 | 说明 |
|------|------|
| 不留大空白 | 结果区未加载时也要有占位(大符号/骨架屏/提示文字),不能是一大片空白 |
| 空状态设计 | "👆 输入XX后自动计算" / "📦 暂无数据" / 大图标 + 提示 |
| 加载状态 | 骨架屏(skeleton)或滚动动画,不是"白屏等结果" |
| 间距合理 | 区块间 16-20px,字段间 8-12px,不要稀疏也不要拥挤 |

#### 3.8.3 配色与层次

| 层次 | 视觉表现 |
|------|---------|
| 主操作 | 渐变背景 + 阴影(主按钮/主结果) |
| 次操作 | 白底 + 主色边框(次按钮/Tab) |
| 内容区 | 浅灰底 var(--color-bg) |
| 强调 | 主色文字/边框(选中态/标签) |
| 错误 | 红底 + 红边 + 红字 |
| 成功 | 绿底 + 绿边 + 绿字 |

**禁止**:
- ❌ 全白底无层次(像未完成的 demo)
- ❌ 单一字号无层级(像记事本)
- ❌ 无任何颜色强调(像 90 年代的 HTML)
- ❌ 按钮和内容区无视觉区分

#### 3.8.4 字号层级(强化)

| 元素 | 字号 | 字重 | 说明 |
|------|------|------|------|
| 舞台主结果 | 48-80px | 700 | 生成器/占卜类核心数字 |
| 核心数字 | 32-36px | 700 | 计算器核心结果 |
| 卡片大数字 | 22-26px | 700 | 明细卡片 |
| 段落标题 | 16-18px | 600 | 区块标题 |
| 字段标签 | 13-14px | 500 | 输入框 label |
| 正文/明细 | 13-14px | 400 | 描述/卡片内容 |
| 提示/hint | 11-12px | 400 | 辅助说明 |

**比例关系**:相邻字号差异 ≥ 4px,层级才清晰。

#### 3.8.5 圆角与阴影

| 元素 | 圆角 | 阴影 |
|------|------|------|
| 舞台/主结果卡 | 14-16px | 渐变自带,无额外阴影 |
| 主按钮 | 10-12px | box-shadow 0 4px 14px |
| 次按钮/Tab | 8px | 无 |
| 明细卡片 | 8-10px | 无 |
| 输入框 | 8-10px | 无(focus 时 border-color 变化) |
| 标签/徽章 | 4-6px | 无 |

**禁止**:圆角不一致(一个页面 3 种以上圆角值)。统一用 CSS 变量或固定值。

#### 3.8.6 动画与反馈

| 场景 | 动画 |
|------|------|
| 结果出现 | scale + opacity 渐入(0.3-0.5s) |
| 生成中 | 滚动数字/摇卦动画(600-800ms) |
| 悬停 | transform: scale(1.05) 或 border-color 变化 |
| 切换 Tab | opacity 渐变(0.2s) |
| 复制成功 | toast 提示 + 按钮短暂变色 |

**禁止**:
- ❌ 无任何动画(像静态截图)
- ❌ 动画过长(>1s,用户等得不耐烦)
- ❌ 动画过短(<0.1s,看不到反馈)

#### 3.8.7 一屏完整度

工具首屏(1080p 桌面)应能完整看到:
1. 标题/Tab
2. 输入区(带默认值)
3. 结果区(带默认结果或占位)
4. 主操作按钮

**禁止**:
- ❌ 首屏只有输入框,结果在第二屏(用户要滚动才能看到)
- ❌ 首屏只有按钮,结果在第二屏
- ❌ 首屏留大片空白,关键内容在第二屏

移动端(375px)可允许垂直滚动,但首屏仍应有"标题+输入+部分结果"。

## 四、交互规范

### 4.1 自动计算

- **所有计算器**:onMounted 时用默认值计算一次,result 一进来就有值
- **输入变化**:@input/@change 触发重新计算,实时更新
- **不需要"计算"按钮**:除非计算很慢(如调API)

### 4.2 可展开区块

- 标题行可点击(全宽 cursor:pointer)
- 右侧"展开▼/收起▲"提示
- 默认收起(明细表/高级功能)
- 展开有过渡动画

### 4.3 Tab切换

- 顶部Tab,切换时自动计算对应模式
- 每个Tab独立的结果区

### 4.4 复制/下载

- 结果区有"📋 复制"按钮
- 文件类有"⬇️ 下载"按钮
- 复制成功有 toast 提示

## 七、问题记录(本次规范沉淀的教训)

### 问题1: 功能改了但界面没变

**现象**: 改了计算逻辑(加年终奖),但用户看到的还是几个输入框
**根因**: 只改了 result 区逻辑,没改首屏可见性(结果需点击才出现)
**修复**: onMounted 自动计算 + 预设默认值 + 结果区始终可见
**规范**: §1.2 第1/2条

### 问题2: 新功能用户看不到

**现象**: 加了还款明细表/提前还款,但用户不知道有这功能
**根因**: 功能藏在 v-if 折叠里,首屏看不到入口
**修复**: 可展开区块的标题行始终可见("📋 还款明细表 展开▼")
**规范**: §1.2 第3条 + §4.2

### 问题3: 日历网格太大一屏看不全

**现象**: 万年历用 aspect-ratio:1,7格变成超大正方形
**根因**: aspect-ratio 强制正方形
**修复**: 改固定行高(52px)+双栏布局
**规范**: §3.3

### 问题4: API名变更只改一处

**现象**: getShengXiao→getAnimal 只改了万年历,漏改生肖
**根因**: 修复时没全局搜索
**修复**: grep -rn 全项目搜索所有调用点
**规范**: 错题集 #014

### 问题5: 触发按钮在结果区上方(2026-07-14 新增)

**现象**: 易经起卦/抽奖/随机数生成等工具,按钮在顶部,结果在按钮下方,用户点击后视线要往下找结果
**根因**: 沿用了"输入→按钮→结果"的工具式布局,但占卜/随机类工具用户预期是"看到结果在视觉中心,点击刷新"
**修复**: 结果展示区作为视觉焦点(顶部+大尺寸+渐变背景),触发按钮放在结果区下方;滚动动画+结果出现动画增强仪式感
**规范**: §1.0 + §3.6

### 问题6: 转换类工具需要点"转换"按钮(2026-07-14 新增)

**现象**: base64/morse 工具输入后需要点"转换"按钮才显示结果
**根因**: 沿用了"触发式"设计,而 JSON/URL/CSS 格式化工具已经是实时转换
**修复**: 移除"转换"按钮,改@input实时转换,onMounted 用默认值转换一次
**规范**: §3.7

### 问题7: 生成器工具按钮上方留空白(2026-07-14 新增)

**现象**: 随机密码/UUID 等生成器,页面顶部是大片空白,生成按钮在中部,用户点击后密码出现在按钮上方但原本是空的,视线焦点散乱
**根因**: 没有把"结果舞台"作为永远占位的视觉焦点,空状态没有占位元素
**修复**: 顶部建一个永远占位的结果舞台,空状态有大符号+提示(比按钮还显眼),按钮在舞台下方,点击后结果在原位刷新
**规范**: §3.6 + §3.8.2

### 问题8: 工具首屏没有视觉重心(2026-07-14 新增)

**现象**: 多个工具首屏是"几个空输入框 + 一个按钮",像未完成的 demo
**根因**: 沿用表单式设计,没有把工具当产品做
**修复**: 每个工具首屏必须有一个最显眼的元素(大数字/舞台/卡片网格),见 §3.8.1 视觉重心对照表
**规范**: §3.8

## 八、检查清单(每个工具上线前)

### 产品设计
- [ ] 进页面就有默认值和结果(不需要先填再点)
- [ ] 所有功能在首屏可见或有明确"可展开"入口
- [ ] 有可视化(进度条/占比条/对比图)
- [ ] 对标了成熟产品的功能
- [ ] 数据量达标(谜语100+/成语500+)
- [ ] **触发按钮不在结果区上方**(生成/占卜/抽奖类:舞台是焦点,按钮在下方) — v2.1
- [ ] **转换类工具不需要"转换"按钮**(实时转换,输入即出结果) — v2.1
- [ ] **首屏有视觉重心**(大数字/舞台/卡片网格,不是"几个空输入框+按钮") — v2.2
- [ ] **空状态有占位**(大符号+提示/骨架屏,不是大片空白) — v2.2
- [ ] **一屏能看完关键内容**(标题+输入+结果+按钮,不在第二屏) — v2.2

### 审美
- [ ] 有渐变主卡(不是全白底无层次) — v2.2
- [ ] 字号层级清晰(相邻字号差 ≥ 4px) — v2.2
- [ ] 圆角统一(一页不超过 3 种圆角值) — v2.2
- [ ] 有动画反馈(结果出现/悬停/切换,0.2-0.5s) — v2.2
- [ ] 颜色语义正确(绿=成功/红=错误/黄=警告/紫=主操作) — v2.2

### 交互
- [ ] 输入变化实时计算(@input/@change)
- [ ] 可展开区块有"展开▼/收起▲"提示
- [ ] 多模式有Tab切换
- [ ] 结果可复制/下载

### 视觉
- [ ] 核心数字醒目(大字+渐变背景)
- [ ] 颜色语义正确(绿=到手/红=支出)
- [ ] 移动端响应式(1列布局)
- [ ] 字号层级清晰

### 规范
- [ ] 不用 aspect-ratio:1 做网格
- [ ] CSS 变量引用(不硬编码色值)
- [ ] ToolLayout 包裹
- [ ] 数据外置 data.ts(不内嵌大数组)
