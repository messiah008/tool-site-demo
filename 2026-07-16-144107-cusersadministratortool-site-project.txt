  ---
  知鸟工具箱 (tool-site-project) 全景梳理

  一、这是什么

  面向中文用户的在线工具站,商业模式是"免费工具引流 + 付费转换":

  - 50+ 免费小工具 引流(SEO/GEO 流量)
  - 高质量 PDF 转 Word 付费服务(WPS 引擎 + OCR,差异化卖点)
  - 卡密兑换 机制对接闲鱼/拼多多电商渠道(线下收款 → 卡密发卡)

  战略上不自建,而是 Fork Stirling-PDF + WPS Worker 增强 来缩短周期。

  二、技术栈

  ┌─────────┬────────────────────────────────────────────────────────────────┐

  三、核心工程理念:一致性链接(SSOT)

  这是整个项目的灵魂,也是 CLAUDE.md 反复强调的红线:

  所有变更从 schema 开始,不直接改代码。四大单一事实来源:

  ┌────────────┬─────────────────────────────┬───────────────────────────────┐
  │   Schema   │            路径             │             控制              │
  ├────────────┼─────────────────────────────┼───────────────────────────────┤
  │ 设计令牌   │ schemas/design-tokens.json  │ 视觉一致性 → CSS 变量         │
  ├────────────┼─────────────────────────────┼───────────────────────────────┤
  │ 工具注册表 │ schemas/tools.registry.json │ 工具一致性(110 条)→ 路由/卡片 │
  ├────────────┼─────────────────────────────┼───────────────────────────────┤
  │ API 契约   │ schemas/api/openapi.yaml    │ 接口一致性 → 前后端类型       │
  ├────────────┼─────────────────────────────┼───────────────────────────────┤
  │ AI 规范    │ CLAUDE.md                   │ 上下文一致性                  │
  └────────────┴─────────────────────────────┴───────────────────────────────┘

  核心循环:识别需求 → 改 Schema → 生成代码 → 实现业务 → 本地校验(verify_all.py

  三个重大发现

  1. audio-convert 是彻底的假工具——Stirling 259
  个端点里没有任何音频/视频转换。你 registry 挂的 backend=stirling 的
  audio-convert 根本无对应端点，得用独立 FFmpeg worker。这也印证了 H.264
  那条线——Stirling 本就不碰音视频。
  2. excel-to-pdf/ppt-to-pdf 实测可转——靠 Stirling 自带的
  unoserver(LibreOffice)，且这个 latest 镜像已经内置，不用换 full
  版。我之前担心的"需额外装 LibreOffice"是多余的。
  3. pdf-ocr 当前 500——ocrmypdf 报错，需查 tesseract
  中文语言包。这是你唯一付费的 stirling 工具，得修好。

  额外发现的"白送"端点（你 registry 没有、但 Stiring 现成能转）

  实测通过、可直接做新工具的：pdf-to-text(✅)、html-to-pdf(✅)、markdown-to-pdf(
  ✅)、pdf-to-html(✅)、pdf-to-epub(✅)、url-to-pdf、svg-to-pdf、pdf-to-xml、pdf
