## 二、huangli.com 设计精髓(对标提取)

| 设计要点 | huangli 做法 | 我们差距 |
|---------|------------|---------|
| **先给结论再给依据** | 首屏大卡:宜忌(9项)左右分栏 → 再列吉神/凶煞/方位 | 万年历详情已有宜忌,但吉神凶煞方位全缺 |
| **时辰级吉凶** | 12时辰表格,每行:时刻/干支/星神/冲/吉凶/五行/煞方/时宜忌 | 完全没有(最大差距) |
| **方位信息** | 财/福/喜神方位 + 胎神占位,单字圆形图标(财福喜) | 完全没有 |
| **彭祖百忌** | "甲不开仓 申不安床" | 完全没有 |
| **反查吉日** | "宜嫁娶吉日/忌入宅吉日"按事项全月反查 | 黄道吉日已有(事项筛未来N天),但无"全月列表"形式 |
| **民俗兆象** | 眼跳/耳鸣等12类时辰吉凶 | 没有(可选,内容工作量大) |
| **今日卦象** | 每日一卦+7维释义 | 没有(可选,需 iching 数据已有) |
| **视觉权威感** | 古典纸质质感/竖排农历/标签分层 | 纯白卡片,缺乏"老黄历"氛围 |

## 三、迭代方案(分优先级)

### 🔴 P0:黄历三件套重做(对标 huangli.com)

#### 3.1 万年历 lunar-calendar —— 加详情深度(改详情右栏)

**现有**: 宜忌/干支/生肖/星座/星宿/冲煞(8项) ✅ 已做
**新增详情模块**(右栏向下扩展,数据全部 lunar-javascript 已验证可用):

```
┌─────────────────────────────┐
│ 农历/干支/生肖/星座(已有)      │
│ 宜 ▓▓▓▓▓ / 忌 ▓▓▓▓▓ (已有)   │
├─────────────────────────────┤
│ 🧭 今日方位                   │  ← 新增
│ 财:东北 喜:东北 福:正北       │   getDayPositionCaiDesc/XiDesc/FuDesc
├─────────────────────────────┤
│ ✨ 吉神宜趋                   │  ← 新增
│ 天德 月德 阳德 司命...(标签)   │   getDayJiShen
│ ⚠️ 凶煞宜忌                   │
│ 劫煞 天贼 五虚...(标签)        │   getDayXiongSha
├─────────────────────────────┤
│ 📜 彭祖百忌                   │  ← 新增
│ 甲不开仓财物耗散               │   getPengZuGan
│ 申不安床鬼祟入房               │   getPengZuZhi
├─────────────────────────────┤
│ ⏰ 十二时辰吉凶(可折叠)         │  ← 新增(核心)
│ 子 23-1 吉 | 宜祈福... 忌赴任  │   getTimeGan/Zhi/Yi/Ji/Chong/Sha
│ 丑 1-3 吉 | ...               │   12行遍历,标记今日当前时辰
│ ...                           │
└─────────────────────────────┘
```

**视觉**: 吉神绿色标签/凶煞红色标签;财福喜用 emoji 圆形图标(💰喜❤️福🍀);时辰表当前时辰高亮。
**交互**: 点时辰行展开详细宜忌;顶部"今日"时辰自动定位。
**工时**: ~4h(纯前端,lunar API 齐全)
**验收**: 详情页信息密度对齐 huangli 首屏;移动端可折叠不臃肿。

#### 3.2 老黄历 old-almanac —— 与万年历统一(避免重复)

**问题**: 老黄历是"输入查单日",万年历是"月历点选",功能重叠,且老黄历信息更少。
**方案**: 老黄历改为"单日详情大卡版"(复用万年历的详情逻辑),适合直接搜"今天黄历"的用户(不想看月历)。
- 即:old-almanac = 万年历详情区 + 日期切换器(无月历),专攻"查某天"
- 万年历 = 月历 + 同款详情,专攻"翻日历浏览"
- **共享一个详情子组件** `AlmanacDetail.vue`,放 `frontend/src/tools/_shared/`(新),两边复用,SSOT 详情逻辑
**工时**: ~3h(含抽共享组件)

#### 3.3 黄道吉日 auspicious-day —— 加"时辰"维度

**现有**: 选事项+日期范围 → 列出宜该事项的吉日列表
**新增**: 每个吉日行展开显示"当天吉时"(时辰干支+吉凶),不只是"整天宜"
**工时**: ~2h

#### 3.4 视觉氛围(可选,P0 锦上添花)
- 详情区加古纸质感背景(浅米色 + 边框纹饰),呼应老黄历
- 农历日期竖排(已有,保留)
- 宜忌用大字+底色块强化(已有)

### 🟠 P1:计算类可视化(低成本高体验)

| 工具 | 现状 | 优化 |
|------|------|------|
| 房贷 mortgage-calculator | 纯数字结果 | 加"还款曲线图"(本金/利息占比随时间,简单折线/堆叠条) |
| 退休年龄 retirement-age | 纯文字 | 加"生涯时间轴"(出生→工作→退休→今,横向时间条) |
| 个税 tax-calculator | 纯数字 | 加"五险一金/个税占比饼图" |

**工时**: 各 ~2h(用现有 CSS 或轻量 SVG,不引图表库)
**原则**: 不引 ECharts 等重库(SSG 体积敏感),用纯 CSS/SVG 实现简单可视化

## 四、⚠️ 必须先修的 bug(zodiac getAnimal 回归)

zodiac/index.vue line 69 又用 `getAnimal()`,但 data.ts 的 key 是生肖(鼠牛虎…猪)。
`getAnimal()` 返回二十八宿动物(2026→"狼"),`zodiacData["狼"]` undefined → fallback `zodiacData['鼠']`(line 70)。
**后果**: 任何年份都显示"鼠",静默 bug。违背错题集 #015。
**修复**: line 69 改 `getYearShengXiao()`(已验证返回正确生肖 key)。grep 全项目确认无其他 getAnimal 残留。

## 五、执行顺序建议(按 ROI)

1. **先修 zodiac getAnimal bug**(10分钟,防线上生肖显示错误)
2. **万年历加详情深度**(P0 核心,对标 huangli 首屏,4h)

## 六、不做的事(避免过度)
- 不引 ECharts/图表重库(SSG 体积 + 月成本敏感,纯 CSS/SVG 足够)
- 不做民俗兆象/P0 范围外的内容(工作量大,先验证 P0 流量效果)
- 不加算命导流(huangli 靠付费算命变现,我们是免费工具+PDF付费,变现路径不同)
