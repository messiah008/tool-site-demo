**收益**: 用户能秒级找到工具,降低跳出。

### P0:占位工具标记 + 收集需求

**问题**: 21 个占位工具点进去是"开发中",浪费点击。
**方案**:
1. 首页卡片加角标区分:

**收益**: 降低挫败感 + 收集真实需求(知道哪些占位工具值得优先做)。
**工作量**: 2 小时

### P0:工具互链扩展(bmcx 模式)

**问题**: 当前 ToolLayout 的"相关工具"只显示 6 个同类,SEO 权重传递弱。

**方案**: 扩展为 bmcx 模式(22 号文档 5.3):
- 同类 35 个 + 随机 35 个 = 每页 70 个互链
- 分两区:"同类工具"+"猜你需要"

```typescript
// ToolLayout.vue 相关工具
const relatedTools = computed(() => {
  const sameCat = activeTools.filter(t => t.category === tool.category && t.id !== tool.id).slice(0, 35)
  const others = activeTools.filter(t => t.id !== tool.id && !sameCat.includes(t))
    .sort(() => Math.random() - 0.5).slice(0, 35)
  return { sameCat, others }
})
```

**收益**: SEO 权重传递 + 用户发现更多工具 + 降低跳出。
**工作量**: 1 小时

### P1:首页分类折叠 + 热门工具

**问题**: 96 个卡片平铺,首页过长。

**方案**:
1. 顶部"热门工具"区(优先级 ★★★ 的 22 个,网格展示)
2. 下方按分类折叠(默认收起,点击展开)
3. 搜索框突出(占据首屏中心)

```
┌─────────────────────────────┐
│     🔍 搜索 96 个工具...      │
├─────────────────────────────┤
│ 🔥 热门工具(22 个)            │
│ [卡片网格]                   │
├─────────────────────────────┤
│ 📄 PDF(11) ▼  🖼️ 图片(5) ▼  │
│ 📝 文本(31) ▼ 🔄 转换(5) ▼   │
└─────────────────────────────┘
```

**收益**: 首屏聚焦热门,减少滚动。
**工作量**: 2 小时

### P1:最近使用 + 收藏(localStorage)

**问题**: 回访用户每次重新找工具。

**方案**:
- 每次进入工具页,记录到 localStorage(`recent_tools`)
- 首页顶部显示"最近使用"(最多 8 个)
- 工具页加"★ 收藏"按钮,收藏的工具置顶

**收益**: 提高回访效率,增加留存。
**工作量**: 2 小时

---

### P1:余额不足友好提示

**问题**: 当前余额不足直接 `location.href = '/redeem'`,突兀。

**方案**: 改为弹窗确认 ✅ 已实施(2026-07-07):

```vue
<el-dialog>
  💎 高质量转换需要消耗余额
  当前余额:0 次
  [去兑换卡密] [取消]
</el-dialog>
```

**收益**: 用户有选择权,不强制跳转,体验更柔和。
**工作量**: 1 小时

---

### P2:移动端适配验证 + 优化

**问题**: 未在移动端验证。

**方案**:
1. Chrome DevTools 移动端模式逐个检查 75 个工具页
2. 常见问题:表格横向溢出、按钮过小、输入框未撑满
3. 统一移动端断点(@media max-width: 768px)

**收益**: 移动用户(预计占 40%+)体验正常。
**工作量**: 半天验证 + 2 小时修复

---

### P2:工具页使用说明(可折叠)

**问题**: 部分工具(如 AES 加密、正则测试)新用户不会用。

**方案**: 工具页顶部加"使用说明"折叠区:
- 默认收起
- 点击展开,显示步骤说明(从 registry.about.how_to 或 seo.how_to)

**收益**: 降低学习成本。
**工作量**: 1 小时(已有 how_to 数据,渲染即可)

---

### P2:转换任务进度百分比

**问题**: 付费转换工具只显示"转换中",无进度。

**方案**: 后端 Worker 上报 progress 到 Redis task Hash,前端轮询显示:
```
🔄 转换中... 65%
████████░░░░ 65%
```

**收益**: 用户有预期,降低焦虑。
**工作量**: 后端 Worker 改造(每阶段更新 progress)+ 前端进度条,3 小时

### P3:工具评分 + 反馈

**方案**: 每个工具页底部加"这个工具有用吗?👍 👎",数据存 Redis。
**收益**: 收集用户反馈,指导优化优先级。
**工作量**: 2 小时

### P3:暗色模式

**方案**: design-tokens 已支持(themes.light/dark),加切换按钮。
**收益**: 现代用户期望,夜间使用友好。

## 六、选品重新评估(v2.0,2026-07-08)

### 6.1 问题诊断

**核心问题**: 当前 75 个已实现工具中,大量是"开发者工具"(JSON/正则/Base64/进制转换/UUID/HMAC/AES),这些面向程序员,受众极小;而 bmcx 的高流量工具是"生活工具"(退休年龄/老黄历/简繁互转/个税/房贷/快递查询),面向 14 亿普通网民。

**数据对比**:

| 维度 | 我们 | bmcx 高流量 | 差距 |
|------|------|------------|------|
| 开发者工具占比 | ~40%(30/75) | ~5% | 过高 |
| 生活工具占比 | ~20% | ~60% | 过低 |
| 付费工具 | 4 个(PDF/证件照/压缩/简历) | 少 | 合理 |
| 高频搜索词覆盖 | 6/25(24%) | - | 严重不足 |

**bmcx 高需求而我们缺失的 17 个工具**:
老黄历、公历转农历、简繁互转、汉字转拼音、个税计算器、房贷计算器、存款利息计算、邮编区号、车牌号查询、天气预报、快递查询、货币汇率、英汉词典、食物卡路里、安全期计算、预产期计算、亲戚计算器

### 6.2 选品问题根因

| 根因 | 表现 | 影响 |
|------|------|------|
| 从开发者视角选品 | 先做了 JSON/正则/Base64(自己用得上) | 受众窄,流量低 |
| 缺乏搜索量验证 | 没查百度指数就定优先级 | 做了低搜索量的工具 |
| 开发者工具优先级虚高 | json-format/regex-test 标 ★3,但搜索量远低于"老黄历" | 资源错配 |
| 生活类工具被忽略 | 个税/房贷/退休年龄没有 | 错过最大流量入口 |
